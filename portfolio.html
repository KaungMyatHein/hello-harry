The Problem

The existing return process was fragmented across 5 different touchpoints, causing customer frustration and a 34% abandonment rate. Support tickets related to returns consumed 40% of the CS team's bandwidth, costing the business significantly.

My Process
  • Stakeholder interviews with CS, warehouse, and finance teams
  • Customer journey mapping across all return touchpoints
  • Competitive analysis of 8 leading retail return experiences
  • Iterative prototyping with 3 rounds of usability testing
  • A/B tested final design against legacy flow
The Solution

Designed a unified self-service return portal with intelligent reason categorization, real-time status tracking, and automated refund processing. Reduced the 5-step fragmented flow to a streamlined 2-step experience with clear visual feedback at each stage.

Fintech Design System 2024

Multi-Product Design System & POS Ecosystem

Built a comprehensive design system from scratch for Raytail's POS ecosystem, achieving 200% UX ROI and $10K monthly savings.

200%
UX ROI
$10K
Monthly Savings
View Full Case Study
The Problem

Raytail's multi-product POS ecosystem had no unified design language. Each product team built UI independently, causing visual inconsistency, duplicated effort, and a fragmented user experience across merchant-facing touchpoints. Dev velocity suffered from recreating components.

My Process
  • Audited UI across 4 products — cataloged 180+ inconsistencies
  • Conducted interviews with 3 dev teams and 2 product managers
  • Built token architecture (color, spacing, typography, elevation)
  • Created atomic components with documented usage guidelines
  • Ran WCAG accessibility audit — achieved AA compliance
  • Phased rollout with adoption tracking across teams
The Solution

Created Raytail's first unified design system with 60+ atomic components, a robust token architecture, and comprehensive documentation. The system bridged Figma and code with a shared component library, enabling teams to ship consistent UI 3x faster.
